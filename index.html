2-1 CLAUDE.md — 프로젝트 영구 지침 파일

📄
한 줄 정의
CLAUDE.md = "Claude Code가 프로젝트에서 매 세션 자동으로 읽는 설정 파일"
한 번 작성하면 팀 전체에 영구 적용됩니다.

어떨 때 써요 — 코드 프로젝트에서 "React 쓰지 마라", "커밋 메시지는 한글로", "API 키 하드코딩 절대 금지" 같은 팀 규칙을 Claude가 매번 지키게 하고 싶을 때.
뭘 할 수 있나요 — 기술 스택·코딩 규칙·팀 컨벤션을 이 파일 하나로 고정합니다. git에 커밋하면 팀원 전체가 동일한 규칙의 AI를 갖게 됩니다.
위치적용 범위Git 공유
~/CLAUDE.md내 모든 프로젝트개인용
프로젝트/CLAUDE.md이 프로젝트만팀 공유
서브폴더/CLAUDE.md이 폴더에서만개인용
CLAUDE.md — 프로젝트 설정 예시 
# 프로젝트 설정

## 기술 스택 (변경 불가)
- 런타임: Node.js + Express
- DB: SQLite → RDS MySQL
- 절대 사용 금지: React, Vue, Tailwind

## 코딩 규칙
- 주석 최소화
- 보안: helmet + express-rate-limit

## 팀 컨벤션
- 커밋: [feat/fix/refactor]: 한 줄 요약
- 시간 표기: KST 기준
💡
앱 Projects와 차이
파일로 존재 → git으로 팀 전체 공유 가능
CLAUDE.md 없으면 Claude가 매번 추측으로 기술 선택 → 이 파일이 있으면 일관성 보장

어떻게 시작해요 — 프로젝트 루트에 CLAUDE.md 파일 생성 → 기술 스택·금지 항목·팀 컨벤션 작성 → git commit → 팀 전체 적용 완료.

사람이 없어도 규칙이 자동 실행됩니다 — 위험 명령어 차단, 저장 후 Lint 자동화, 완료 시 Slack 알림까지 모두 Hook 하나로 설정.

2-3 Hooks 심화 — 설정 파일로 규칙 자동화

⚠️
Hook이 없으면 어떻게 될까?
Claude가 실수로 rm -rf /를 실행해도 막을 방법이 없습니다. PreToolUse Hook에 위험 패턴 감지 스크립트 하나만 달면 실행 전에 자동 차단됩니다.

어떨 때 써요 — AI가 코드를 직접 수정·삭제·배포하는 프로젝트에서. 팀 표준 도구(Lint·테스트·알림)를 사람 개입 없이 자동 실행하고 싶을 때.
뭘 할 수 있나요 — 설정 파일 한 번만 작성하면 이후 모든 세션에서 자동 작동합니다. 위험 차단부터 Slack 알림까지 전부 코드로 강제됩니다.
.claude/settings.json — Hooks 설정 예시 
"hooks": {
  "PreToolUse": [{
    "matcher": "Bash",
    "hooks": [{
      "type": "command",
      "command": "echo $CLAUDE_TOOL_INPUT | grep -i 'rm -rf' && echo '🔴 위험 명령어 감지!' || true"
    }]
  }],
  "Stop": [{
    "hooks": [{
      "type": "http",
      "url": "https://hooks.slack.com/your-webhook-url",
      "method": "POST"
    }]
  }]
}
💡 설정 위치
.claude/settings.json 파일에 "hooks" 키를 추가합니다. Claude Code를 재시작하면 즉시 적용됩니다.
핸들러 타입하는 일대표 활용
command쉘 명령어 실행Lint·테스트·파일 로그 기록
httpURL로 POST 웹훅Slack·Discord·Teams 알림
promptClaude에 추가 지시삭제 전 "정말 실행?" 확인
agent서브에이전트 실행완료 후 자동 코드 리뷰
🔧 Hook 환경 변수 전체 보기 (스크립트 작성 시 참고)
Hook에서 사용 가능한 환경 변수
$CLAUDE_TOOL_NAME
→ 실행된 도구 이름 (예: Bash, Edit, Write)
$CLAUDE_TOOL_INPUT
→ 도구에 전달된 입력값 (명령어 내용 등)
$CLAUDE_TOOL_RESULT
→ 도구 실행 후 결과값
💪 직접 해보기 — 작업 완료 알림 Hook 추가하기
💪 직접 해보기
작업 완료 알림 Hook 추가하기
① .claude/settings.json 열기 ② "hooks":{"Stop":[{"hooks":[{"type":"command","command":"echo 완료!"}]}]} 추가 ③ Claude Code 재시작 ④ 작업 지시 후 알림 확인

2-4 Subagents — 독립 AI 에이전트 위임

🤖
한 줄 정의
Subagent = "독립 컨텍스트에서 전문 작업을 수행하는 하위 AI"
메인 에이전트가 지휘하고, 서브에이전트가 전문 실행합니다.

어떨 때 써요 — 코드 리뷰·보안 감사·문서 작성을 동시에 해야 할 때. 또는 작업이 너무 커서 하나의 Claude 컨텍스트로 감당 안 될 때.
뭘 할 수 있나요 — 3개 작업을 순서대로 하면 3배 걸리지만, 서브에이전트 3개가 동시 실행하면 같은 시간에 전부 완료됩니다.
특징설명
독립 컨텍스트메인 대화 오염 없이 대용량 작업 처리
병렬 실행여러 에이전트 동시 실행 → 속도 향상
최소 권한역할별 필요한 도구만 부여 → 안전성
격리 모드isolation: "worktree" → 안전한 실험
💪 직접 해보기 — 서브에이전트 정의 파일 작성
💪 직접 해보기
서브에이전트 정의 파일 작성
.claude/agents/code-reviewer.md 작성 → 코드 리뷰 전문 에이전트 설계 → 실제 파일로 테스트
Agent 도구 호출 예시 
// 단일 에이전트
Agent({
  description: "코드 리뷰",
  prompt: "auth.js 보안 검토",
  subagent_type: "code-reviewer"
})

// 병렬 에이전트 (동시 실행)
Agent({ prompt: "프론트엔드 분석" })
Agent({ prompt: "백엔드 분석" })
Agent({ prompt: "DB 스키마 분석" })
// → 3개 동시 실행, 결과 취합
⚡
병렬 처리
독립 작업을 동시 실행 → 3배 빠른 처리.
예: 프론트·백·DB 분석을 한 번에.
🔒
격리 실행
worktree 모드 → 실험 중 실수가 메인 코드에 영향 없음.
