Custom Instructions 작성 예시

Claude 앱 Projects → Custom Instructions 예시 
## 우리 팀 글쓰기 규칙
- 모든 보고서: "결론 먼저, 근거 나중" 구조
- 문장은 2줄 이내로 끊기
- 영어 약어 첫 등장 시 한국어 풀이어 병기
  예: LMS (학습관리시스템), ROI (투자대비효과)

## 금지 표현
- "~한 것 같습니다" 금지 → "~입니다"로
- "많은" 금지 → 구체적 숫자로
- 번역투 금지: "~함에 따라", "~의 관점에서"

## 직급·호칭 체계
- 팀장: OO 팀장님 / 실장: OO 실장님 / 대표: 대표이사님
📝
보고서 팀
"우리 팀 보고서 기준서 + 용어집" 업로드 → 모든 보고서 초안에 자동 반영. 매번 설명 불필요.
🏢
임원 보고 준비
"사업 계획 문서 전체" 업로드 → 임원 Q&A 준비용 Claude 즉시 참조. "이 사업의 3대 리스크는?" 물으면 문서 기반 답변.
👥
팀 공유
Team/Enterprise 플랜: 팀원 모두 같은 Project 사용 → 팀 전체 일관된 AI 응답. 개인 설정 불필요.

2-1  CLAUDE.md — 프로젝트 영구 지침

📄
한 줄 정의

계층 구조 — 상위가 하위를 오버라이드

CLAUDE.md — 프로젝트 설정 예시 
# 프로젝트 설정

## 기술 스택 (변경 불가)
- 런타임: Node.js + Express
- DB: SQLite → RDS MySQL 순차 전환
- 절대 사용 금지: React, Vue, Tailwind, SCSS, PostgreSQL

## 코딩 규칙
- 주석 최소화 — 코드 자체가 설명되도록
- 에러 핸들링은 시스템 경계(입력·외부 API)에만
- 보안: helmet + express-rate-limit 필수

## 팀 컨벤션
- 커밋 메시지: [feat/fix/refactor]: 한 줄 요약
- PR 전 E2E 브라우저 테스트 필수
- 모든 시간 표기: KST 기준
위치적용 범위용도
~/CLAUDE.md내 모든 프로젝트개인 전역 규칙 (선호 언어·출력 형식)
프로젝트/CLAUDE.md이 프로젝트만팀 공통 규칙 (git으로 공유)
서브폴더/CLAUDE.md이 폴더에서만모듈별 특수 규칙
💪 핸즈온 실습 — 15분
자기 팀 프로젝트용 CLAUDE.md 초안 작성
실제 내 프로젝트 폴더에서 → touch CLAUDE.md → 기술 스택·코딩 규칙·팀 컨벤션 3가지 섹션 작성 → claude 실행해서 Claude가 읽었는지 확인
🎤 강사 노트
  • CLAUDE.md가 앱 Projects Custom Instructions와 다른 점: 파일로 존재하므로 git으로 관리·공유 가능
  • "전역 ~/CLAUDE.md에 회사 전체 공통 규칙, 프로젝트별 CLAUDE.md에 개별 규칙" — 계층 활용 예시 꼭 보여주기
  • CLAUDE.md 없으면 Claude가 매번 추측으로 기술 선택 → 이 파일이 있으면 일관성 보장

2-2  Skills (.claude/skills/) — CLI 전용 스킬

💡
앱 Skills와 무엇이 다른가?
앱 Skills는 claude.ai GUI에서 관리. CLI Skills는 .claude/skills/ 폴더에 파일로 존재 → git으로 팀 전체에 공유 가능. 슬래시 명령어도 자동 생성됩니다.
.claude/skills/ — 폴더 구조 
.claude/skills/
├── code-review/
│   ├── SKILL.md           ← 코드 리뷰 지침
│   └── checklist.md       ← 부속 체크리스트
├── deploy-guide/
│   ├── SKILL.md           ← 배포 가이드
│   └── rollback.md
└── report-draft/
    └── SKILL.md           ← 보고서 초안 생성

2-3  Hooks — 워크플로우 자동화·규칙 강제

🪝

"이 파일 삭제 안전한가?"
agent (실험적)

코드 리뷰 에이전트 자동 호출

설정 파일 위치별 적용 범위

위치적용 범위Git 공유

.claude/settings.local.json이 프로젝트, 나만❌ 개인용
이벤트발생 시점주요 활용

PostToolUse도구 실행 후Lint 자동 실행, 포맷터 적용

.claude/settings.json — Hooks 설정 예시 
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "npx eslint --fix $CLAUDE_TOOL_OUTPUT_FILE"
      }]
    }]
  }
}
